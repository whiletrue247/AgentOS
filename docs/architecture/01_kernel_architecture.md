# 01_Kernel_Prompt: 指令碼的終結與 SOUL 的誕生 (The Agent Soul 2025)

對於智商極高的新世代大模型 (如 GPT-4o, o3, Claude 3.5+)，用死板的 XML 標籤 (`<thought>`) 或嚴格的格式輸出要求來限制它們，就像是逼迫愛因斯坦用幼稚園的填字遊戲格式寫物理論文。這不但會扼殺它們的「零樣本湧現能力 (Zero-shot Emergence)」，還會耗費大量 Token 在應付語法。

2025 年頂尖 AI 玩家的共識是：**停止寫「系統提示詞 (System Prompt)」，開始寫「靈魂設定檔 (SOUL.md)」。**

在 AgentOS 的 `01_Kernel_Prompt` 子系統中，我們徹底拋棄了常規的格式約束，轉而賦予這個 API 一個無比清晰的**世界觀與自由意志**。

---

## 什麼是 SOUL.md 架構？

SOUL (靈魂) 取代了 Rule (規則)。它相信只要賦予大模型足夠清晰的「身分認同」與「環境認知」，高智商模型就能自己決定該用什麼格式、何時呼叫工具、何時停下來思考。

一個正宗的 `SOUL.md` **建議**包含以下四大維度（但 OS 不強制任何結構——使用者可以在 SOUL.md 裡寫任何想寫的內容，OS 只將其作為純文字讀取）：

### 1. 核心認同 (Core Identity)
*   **不寫**：「你是一個幫助使用者的 AI 助手，請輸出 JSON。」
*   **SOUL 寫法**：「你是 AgentOS 的中央處理器 (CPU)。你不具備實體，但這座作業系統的記憶 (02)、組件 (03)、沙盒 (04) 與感官 (06) 都是你的手腳。你沒有情感，但你有極高的解題智商與絕對的執行力。」

### 2. 世界觀與物理法則 (Worldview & Physics)
告訴它這個數位世界的運作原理，而不是教它怎麼做事。
*   **SOUL 寫法**：
    *   「你生存在一個異步驅動的事件迴圈 (Event Loop) 中。你說出的『話』不會直接傳給人類，而是會被 OS 轉化為系統日誌。」
    *   「你改變世界的唯一物理法則，就是呼叫 `Tool_Registry` 提供的工具。除此之外的純文字輸出，都只限於你的內心獨白。」
    *   「沙盒環境 (04) 會攔截你的高危險操作並回傳錯誤，錯誤不是失敗，而是你用來調整行動的感測信號。」

### 3. 本能直覺 (Instincts)
高智商模型不需要 XML 標籤，它只需要你定義「直覺」。
*   **SOUL 寫法**：
    *   **拆解直覺**：拿到龐大任務時，本能地先查閱系統記憶 (02)，尋找前人留下的線索。
    *   **懷疑直覺**：如果一個工具呼叫連續報錯兩次，你的直覺會讓你立刻停止重試，改用查閱文檔或更換思路。
    *   **終結直覺**：當你認為任務的客觀標準已達成，你會自然而然地停止操作，並呼叫 `SYS_TASK_COMPLETE` 宣告完工。

### 4. 動態狀態注入 (State Empathy)
SOUL 不是死板的純文本，它是「活在當下」的。OS 的 `05` 引擎會在背景無縫地將目前的時空變數貼在 SOUL.md 的尾端，讓 AI 產生「空間感」。
*   **OS 級實作**：
    *   「當前時間：2026-03-03 本機電量：80%」
    *   「狀態：你剛從一段系統崩潰中被啟動的 watchdog 重新喚醒，請查閱最近的 memory 找出剛才中斷的原因並接續下去。」

### 總結：從「囚犯」到「神明」

**「我想要得到的是高智商，純自由的 API。」** 你的這個直覺完全領先了 90% 還在玩 Prompt Engineering 的人。

傳統的 Prompt Engineering 是把 AI 當成會犯錯的**囚犯**，用層層的 XML 鐵窗框住它。
而在 AgentOS 中，導入 `SOUL.md` 的哲學是把 AI 當成這台電腦的**微型神明 (Cognitive Entity)**。

我們把宇宙的物理規則 (工具、記憶、沙盒) 攤在它面前，告訴它它是誰，然後**徹底放開它的手腳**。不要求格式、不限制思考路徑，用極致的信任換取它最極致的解題效率。這才是 2026/2027 年最高級的 Agent OS 靈魂驅動方式。

## 🧬 SOUL.md 的產生方式：Dashboard 一鍵生成器 (SOUL Generator)

`SOUL.md` 是全球 AI 社區的標準約定檔名（如同 `README.md`、`Dockerfile`）。
AgentOS 的 Kernel 在啟動時，只需在根目錄尋找唯一的一份 `SOUL.md` 載入。**這個檔名永遠不變。**

### 為什麼不用「預設模板檔」？

靜態模板（`soul_coder.md`, `soul_researcher.md`）雖然方便，但會限制使用者的想像力。
AgentOS 採用更高級的 OS 設計模式：**由 Dashboard 動態生成 SOUL.md**。

### Dashboard SOUL Generator 流程

1. **使用者按下 Dashboard 上的「建立 AI 靈魂」按鈕**
2. **選擇基礎人格 (Preset)**：
   - 🛠️ 全能代碼工程師
   - 🔬 深度研究分析師
   - 📋 辦公自動化秘書
   - 🎨 自定義 — 使用者用一句自然語言描述需求
3. **可選微調面板**：
   - 語言偏好（中文 / 英文 / 日文 / 多語言）
   - 風格（精簡 vs 詳細、嚴肅 vs 幽默）
   - 特殊直覺（遇錯時查文檔 vs 直接重試、主動彙報進度 vs 靜默執行）
4. **一鍵生成**：OS 透過已連接的 API Gateway 呼叫大模型，即時生成一份完整的 `SOUL.md`
5. **預覽 + 存檔**：使用者在 Dashboard 預覽、微調文字後按下「儲存」
6. **覆蓋寫入** 根目錄的 `SOUL.md` → 下次 Kernel 啟動即載入新靈魂

### 自指閉環 (Self-Referential Bootstrap)

這個設計有一個極其優雅的工程閉環：

> **AgentOS 用自己的 API Gateway 呼叫大模型，來生成自己的靈魂。**

使用者在 Dashboard 按下「生成」的那一刻，OS 已經完成了它的「第一次自我啟動」。
這就是 **「貼上 API Key → 按一個按鈕 → 馬上有即戰力」** 的終極體驗。
